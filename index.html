What projects have I worked on?
  • Redis Tries - This project added a module for the Tries data structure, along with a suite of text-based tools such as spellcheck and fulltext search, to Redis. Redis is an in memory, open-source data structure store that provides its users with implementation for commonly used data structures such as hash maps and array lists. The implemented text tools took advantage a trie's key length insertion and search run times. I primarily worked on implementing the trie data structure in C, creating a wrapper Redis module for it, and closely ineracting with other teams to implement other, often unforeseen but crucial functions for their tools on the trie level. I worked on a small team with five others (with five teams working on this project in total) and we ran through the full software development process, from designing with UML to deploying with docker and AWS, end-to-end. I'm incredibly proud of our end product, how we were able to run efficiently and quickly through the scrum process, and the demo we put together to pitch to a panel of accomplished engineers.
  • The Legal Bullet - The Legal Bullet is a legal startup that seeks to disrupt the legal industry by becoming the "Turbo Tax" of law firms and providing quick and affordable legal services. As a one of the five developers, I built a web platform that serves as a hub for easy form filling for paperwork such as immigration visa applications and LLC formations. I specifically focused on building the REST API, automating the build flow from commit to deployment, and implementing a basic MongoDB backend. The platform is currently in beta testing with a network of private attorney clients that allows them to file visa applications at a fraction of the industry standard.
